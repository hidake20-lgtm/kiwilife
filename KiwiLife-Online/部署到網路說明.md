# 🌐 Kiwi Life Online — 公開部署說明（任何人都能玩）

部署完成後會得到一個 `https://xxx.onrender.com` 網址，傳給任何人即可多人同樂（最多 40 人）。

## 需要準備
- GitHub 帳號（github.com，免費）
- Render 帳號（render.com，免費，可直接用 GitHub 登入）

## 步驟一：把遊戲放上 GitHub（全程用瀏覽器，不用打指令）
1. 登入 github.com → 右上角 ➕ → **New repository**
2. 名稱輸入 `kiwilife`，選 **Public** → **Create repository**
3. 點頁面上的 **uploading an existing file** 連結
4. 把 `KiwiLife-Online` 資料夾裡的檔案**全部拖進去**：
   - `server.js`、`package.json`、`render.yaml`、`.gitignore`、`public` 資料夾（含 index.html 和地圖圖片）
   - ⚠️ **不要**拖 `node_modules` 資料夾（如果有的話）
5. 按 **Commit changes**

## 步驟二：在 Render 部署
1. 登入 render.com（建議直接「Sign in with GitHub」）
2. 點 **New +** → **Web Service**
3. 選擇剛剛的 `kiwilife` repository → **Connect**
4. 設定會自動帶入（已寫在 render.yaml）；若沒有，手動填：
   - Build Command：`npm install`
   - Start Command：`npm start`
   - Instance Type：**Free**
5. 按 **Create Web Service**，等 2-3 分鐘建置完成

## 步驟三：開玩！
- 部署完成後頁面上方會顯示網址，例如 `https://kiwilife-hanmer.onrender.com`
- 把網址傳給朋友，打開就能一起玩（輸入暱稱進入，會看到彼此＋Enter 聊天）

## 📌 注意事項
- **免費方案會休眠**：15 分鐘沒人玩會睡著，下一個人打開要等約 50 秒喚醒（畫面會轉圈，正常）
- **更新遊戲**：之後遊戲改版，把新的 `public/index.html` 重新上傳到 GitHub（repo 頁面 → public 資料夾 → Add file → Upload files 覆蓋），Render 會自動重新部署
- 同 WiFi 測試仍可用 `start.command`（見 啟動說明.md）
